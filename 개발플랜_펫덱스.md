## 1. 기술 스택 (확정)

| 영역 | 선택 | 비고 |
|------|------|------|
| 빌드 | **Vite + React + TypeScript** | 로그인 뒤·카메라 중심 → SEO 불필요 → Next보다 단순/빠름 |
| 스타일 | **Tailwind CSS v4** | `@tailwindcss/vite` 플러그인, CSS-first (tailwind.config.js 불필요) |
| 아이콘 | **Material Symbols** | npm `material-symbols`로 self-host (PWA 오프라인·CSP 안전). 대안 Lucide |
| PWA | **vite-plugin-pwa** | "홈 화면에 추가" = 무료 테스트 배포 |
| 클라 상태 | **Zustand** | 셀러비서Pro와 동일 |
| 서버 상태 | **TanStack Query** | 캐싱/동기화 |
| 라우팅 | **React Router** | |
| 백엔드 | **Supabase** | Auth + Postgres + Storage(고양이 이미지) |
| 누끼(배경제거) | **@imgly/background-removal-web** | 브라우저 무료, 호출당 0원 |
| 등급/스탯 | **규칙기반 엔진(자체)** | deterministic·재현가능 (LLM 아님) |

### 네이티브 전략 (중요)
- **웹/PWA로 먼저 개발 → 추후 Capacitor로 감싸 네이티브(iOS/Android) 출시.**
- 코드 한 벌로 웹 테스트 → 앱스토어까지. 재작성 없음.
- 카메라: 웹은 `getUserMedia`, 네이티브 전환 시 `@capacitor/camera`로 교체(어댑터로 추상화).
- Expo/RN은 "진짜 네이티브"지만 Vite/웹을 버려야 해 비채택.

## 3. 초기 셋업 명령어

```bash
# 1) Vite 프로젝트 (이 폴더에서)
npm create vite@latest . -- --template react-ts

# 2) Tailwind v4
npm i tailwindcss @tailwindcss/vite
#   vite.config.ts 에 tailwindcss() 플러그인 추가
#   src/index.css 최상단:  @import "tailwindcss";

# 3) 핵심 라이브러리
npm i react-router-dom zustand @tanstack/react-query @supabase/supabase-js
npm i @imgly/background-removal
npm i material-symbols
npm i -D vite-plugin-pwa

# 4) 환경변수
cp .env.example .env.local   # Supabase URL/anon key 입력
```

> ⚠️ `npm create vite`가 "디렉터리 비어있지 않음" 경고 시 → 기존 파일 유지 옵션 선택(플랜/스키마 폴더는 그대로 둠).

### 아키텍처 규칙 (셀러비서Pro 계승)
- **타입 먼저**(`types/`) → 서비스/엔진 → UI.
- **등급·스탯 = 규칙기반 엔진**(deterministic). 이미지 생성/LLM은 점수에 관여 안 함.
- **카메라는 어댑터로 추상화**(웹 getUserMedia ↔ Capacitor) → 네이티브 전환 시 UI 변경 0.
- **배경 = 코드(SVG/CSS) 팝아트 템플릿**, 맥락/등급/타입별 색 변형. 생성형 0원. (실제 템플릿은 코딩 단계에서 제작)

## 5. DB 스키마 요약 (`supabase/migrations/0001_init.sql`)

| 테이블 | 설명 |
|--------|------|
| `profiles` | 유저(=auth.users) · xp/level · **churu(미끼)**/coins |
| `cat_cards` | ★ 캐치한 카드: 이미지(원본/누끼) · rarity · tribe · stats(jsonb) · ability · **bg_theme** · 캐치맥락(지역/시각/날씨/계절) · dex_no(일련번호) · intimacy |
| `friendships` | 친구 관계 (냥체인지용) |
| `trades` | 냥체인지: 카드↔카드 / 선물(requested null). **현금 필드 없음(안전선)** |

- enums: `animal_type`(cat/dog/other 확장대비) · `rarity` · `time_of_day` · `weather` · `trade_status`
- RLS: 본인 데이터만. 신규가입 시 profile 자동 생성 트리거.
- Storage: 버킷 `cards`(원본/누끼 이미지) — 코딩 단계에서 정책 추가.
- 추후(미포함): 실물주문·후원·도감세트·RAG embedding.

## 6. MVP 개발 순서 (제안)

1. 셋업(Vite/Tailwind/PWA/Supabase 연결)
2. 타입 정의(`types/`) + 스키마 적용
3. **캐치 PoC**: 카메라 → 누끼 → 규칙엔진(등급/스탯) → cat_cards 저장
4. 도감(Dex) 그리드 + 카드 상세(앞/뒷면)
5. 배경 팝아트 템플릿 1~2종 + 맥락 배정
6. 츄르(미끼)·일일 제한·XP
7. 냥체인지(교환) — 이후
8. Vercel 배포 → 테스터 URL 공유
